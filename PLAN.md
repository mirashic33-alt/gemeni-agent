## Полный список инструментов агента

Это то, что агент будет уметь делать. Реализуются постепенно по эпизодам.

### Файловая система
| Инструмент | Что делает |
|---|---|
| `list_files` | Показать содержимое папки |
| `read_file` | Прочитать файл |
| `write_file` | Создать или полностью перезаписать файл |
| `append_file` | Дописать в конец файла (не трогает существующее) |
| `patch_file` | Точечная замена: найти строку → заменить на новую |
| `create_dir` | Создать папку |
| `delete_file` | Удалить в корзину (не навсегда) |
| `search_files` | Найти файлы по маске или имени |

> **Про patch_file:** критически важный инструмент. Без него модель при редактировании
> больших файлов перезаписывает всё с нуля и теряет контент. Patch позволяет заменить
> только нужный фрагмент, не трогая остальное.

### Shell и процессы
| Инструмент | Что делает |
|---|---|
| `run_shell` | Выполнить PowerShell команду, вернуть результат |
| `run_file` | Запустить .py / .bat / .exe как отдельный процесс |
| `list_processes` | Список процессов, запущенных агентом в этой сессии |
| `kill_process` | Остановить процесс по ID |

### Память и архив
| Инструмент | Что делает |
|---|---|
| `read_memory` | Прочитать конкретный файл памяти (SOUL.md, USER.md и т.д.) |
| `append_memory` | Дописать факт в файл памяти |
| `patch_memory` | Точечно отредактировать файл памяти |
| `search_archive` | Поиск по всем файлам памяти и дневным логам |
| `read_digests` | Прочитать последние N дайджестов |

### Интернет
| Инструмент | Что делает |
|---|---|
| `search_internet` | Поиск через Google (Gemini grounding) |

### Изображения
| Инструмент | Что делает |
|---|---|
| `analyze_image` | Описать картинку, OCR, ответить на вопрос по изображению |
| `generate_image` | Сгенерировать изображение |

## Episode 6 — Интернет

Агент может искать актуальную информацию.

**Архитектура (протестировано апрель 2026, работает):**

```python
tools = [
    types.Tool(google_search=types.GoogleSearch()),       # Google выполняет сам (сервер)
    types.Tool(function_declarations=[...]),              # мы выполняем сами (клиент)
]
config = types.GenerateContentConfig(
    tools=tools,
    tool_config=types.ToolConfig(
        include_server_side_tool_invocations=True  # обязательно, иначе ошибка 400
    )
)
```

**Важно:**
- Требует актуальной версии SDK: `pip install --upgrade google-genai`
- AFC (automatic function calling) будет отключён — это нормально, используем ручной цикл
- Класть оба инструмента в один `Tool` объект НЕ работает — поиск не происходит,
  модель использует внутренние знания вместо реального поиска

- Режимы: авто / всегда / никогда
- Источники в ответе

## Episode 8 — Дайджесты и рефлексия

Агент учится и запоминает важное.

- Автогенерация дайджеста за прошедший день
- Папка `workspace/digests/` — архив
- Извлечение фактов из диалогов → `USER.md` / `MEMORY.md`
- Система ошибок: `ERRORS.md` → `LEARNINGS.md`
- **Silent compaction** (идея из OpenClaw): когда история приближается к лимиту,
  агент делает "тихий ход" — сам решает что важного сохранить в MEMORY.md,
  прежде чем старые сообщения будут обрезаны. Ничего не теряется.
- **STYLE.md** — отдельный файл для манеры речи агента (не личность, а стиль)
- История чата в **JSONL** вместо JSON — каждое сообщение отдельной строкой,
  удобнее для больших архивов

## Безопасность — важное из OpenClaw

У них нашли уязвимость: через внешний контент (сайты, файлы) можно заставить агента
изменить SOUL.md и вписать туда вредоносные инструкции, которые переживают перезапуск.

Для нас это актуально начиная с Episode 6 (интернет) и Episode 7 (файлы извне):
- SOUL.md и системные файлы памяти — только для чтения из инструментов
- Изменять их только по явной команде пользователя, не по содержимому внешних данных
- Не давать агенту писать в системные MD-файлы автоматически на основе прочитанного контента
