# Gemeni Agent — План разработки

> Принцип: один этап — одна проверка. Не переходим дальше, пока текущий шаг не работает.

---

## Структура проекта

```
gemeni-agent/
│
├── main.py                        ← точка входа
├── PLAN.md
├── PROGRESS.md
├── requirements.txt
├── .gitignore
│
├── core/
│   ├── startup.py                 ← фоновая инициализация (QThread)
│   ├── message_worker.py          ← отправка сообщений в фоне (QThread)
│   └── bridge.py                  ← thread-safe мост Qt ↔ asyncio (Telegram)
│
├── llm/
│   └── provider.py                ← клиент Gemini (google-genai)
│
├── tools/
│   ├── time_sense.py              ← контекст даты/времени для промпта
│   └── system_monitor.py          ← CPU, RAM, диск (psutil)
│
├── channels/
│   └── telegram/
│       └── bot.py                 ← Telegram бот (python-telegram-bot v20+)
│
├── ui/
│   ├── main_window.py             ← главное окно
│   ├── settings_dialog.py         ← диалог настроек (ключи → keystore)
│   └── theme_config.py            ← тёмная тема (QSS)
│
├── data/
│   ├── logger.py                  ← логгер с обрезкой (300 строк)
│   ├── chat_history.py            ← история чата (JSON, 100 сообщений)
│   └── keystore.py                ← шифрование ключей через Windows DPAPI
│
└── workspace/                     ← рабочие файлы агента
    ├── agent.md                   ← системный промпт / манифест (заполняется)
    └── diary/                     ← дневные записи агента
```

---

## Этапы

### ✅ Этап 1 — Скелет проекта
- Структура папок, все пакеты с `__init__.py`
- `ui_export.pyw` → `ui/main_window.py`, `theme_config.py` → `ui/theme_config.py`
- `main.py` — единая точка входа
- **Проверка:** `python main.py` → окно открывается

---

### ✅ Этап 2 — Настройки и тема
- `ui/settings_dialog.py` — три поля: Gemini API Key, Telegram Token, Chat ID
- Тема применяется глобально через `QApplication.setStyleSheet()`
- **Проверка:** диалог настроек открывается в тёмной теме

---

### ✅ Этап 3 — Шифрование ключей
- `data/keystore.py` — Windows DPAPI (без паролей, через ctypes)
- Ключи хранятся в `data/keys.enc`, не в открытом `.env`
- **Проверка:** ключи сохраняются и читаются между запусками

---

### ✅ Этап 4 — LLM провайдер и инициализация
- `llm/provider.py` — клиент Gemini (`google-genai`, модель `gemini-3-flash-preview`)
- `data/logger.py` — обрезка до 300 строк, разделитель сессий
- `core/startup.py` — QThread: читает ключи → пингует модель → загружает историю
- Статус-бар обновляется на каждом шаге
- **Проверка:** статус-бар прошёл все шаги, в заголовке — название модели

---

### ✅ Этап 5 — Чат и память сессии
- `core/message_worker.py` — запросы к модели в фоне (QThread)
- `Enter` = отправить, `Shift+Enter` = новая строка
- `client.chats.create(history=...)` — модель помнит всю сессию
- `data/chat_history.py` — JSON, 100 сообщений, восстановление при старте
- Кнопка 🗑 — очистить историю
- **Проверка:** модель помнит предыдущие сообщения после перезапуска

---

### ✅ Этап 6 — Статус-бар и мониторинг
- Левая часть: прогресс → `Думаю... 3.45 сек` (живой таймер 50 мс) → `Готов.`
- Правая часть: `CPU: 23% | RAM: 64% | C: 93.1GB` (psutil, каждые 2 сек)
- **Проверка:** таймер тикает в реальном времени, мониторинг обновляется

---

### ✅ Этап 7 — Telegram бот
- `channels/telegram/bot.py` — polling в отдельном потоке (asyncio)
- Двустороннее дублирование: UI ↔ Telegram
- `core/bridge.py` — Qt-сигналы для thread-safe обмена
- Команды: `/start`, `/clear`
- Typing-индикатор пока модель думает
- **Проверка:** сообщения синхронизируются в обе стороны

---

### ⬜ Этап 8 — Системный промпт из файла
- Читать `workspace/agent.md` при каждом старте
- Передавать в `provider.start_chat(system_prompt=...)`
- Модель сама дополняет файл со временем
- **Проверка:** изменить `agent.md` → поведение модели меняется

---

### ⬜ Этап 9 — Инструменты файловой системы
- `tools/filesystem.py` — `list_dir`, `read_file`, `write_file` для `workspace/`
- Function calling через Gemini API
- **Проверка:** попросить агента показать список файлов в workspace

---

### ⬜ Этап 10 — Системный трей
- `ui/tray.py` — иконка в трее Windows
- Закрытие окна → скрыть (не убивать процесс)
- Меню: «Показать», «Выход»
- **Проверка:** свернуть → найти в трее → развернуть
